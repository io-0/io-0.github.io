Software Entwicklung

Was > Wie

  • Für SW Entwickler spannend ist das Wie

  • Das Was ist jedoch was mehr Gewicht und Bestand hat

  • Wie gehen wir damit um?

Basis Werkzeuge um das Was festzuhalten:

  • Namen

  • Gruppierung (Kontext geben)

  • Kommentare

Von Unten nach Oben..

Typen und Operatoren

  • int

    • das Was sind die ganze Zahlen
    • das Wie ist teil der Sprache

  • +

    • das Was ist Addition
    • das Wie ist teil der Sprache

Von Unten nach Oben..

Funktionen


            int sum(int a, int b, int c) { 
              return a + b + c;
            }

  • unser Was ist das Summieren von ganzen Zahlen

    • idealer Weise ausreichend beschrieben durch die Signatur

  • das Wie steht später als Block und verknüpft simple! Bausteine

Von Unten nach Oben..

Methoden (Klassen)

  • wie Fn aber stärkere Gruppierung über Klasse

  • diese Gruppe teilt sich Kontext, mindestens ein Aufgabengebiet, potentiell aber mehr

  • je mehr geteilt wird desto schwerer ist unser Was zu erfassen, Signatur reicht nicht mehr

  • das Wie verknüpft Bausteine die einen aus Block-Sicht unkontrollierten Zustand haben können

    • "Globale Variablen", Side Effects, FP rules :D

Von Unten nach Oben..

Interface


            int sum(int a, int b, int c);

  • das Was steht im Vordergrund

    • idealer Weise ausreichend beschrieben durch die Signaturen

  • diese Gruppe teilt sich ein Aufgabengebiet

  • das Wie ist nebensächlich, kann mehrere verschiedene geben

Von Unten nach Oben..

Layer & Packages


            // Controller, Service, Repository, Utils, ..

  • Orientierungshilfen wo man Was findet

Von Unten nach Oben..

Test

  • ist Funktionsnachweis, Was Orientiert

  • guter Platz um Features / Ziele / Verhalten (das Was) zu dokumentieren

    • Testabdeckung wird zum Dokumentationsindikator

  • nutzbar als Aufgaben Verständnis Prüfer

  • wird zu Software Funktions Nachschlagewerk

  • dokumentiert Zeit & Budget Kompromisse

    • man kann abgrenzen was behandelt/bedacht wurde, festhalten in welchen Usecases es definiertes Verhalten gibt

Von Unten nach Oben..

Logs

  • essenzielles Hilfsmittel beim Nachvollziehen von Was passiert ist (undefiniertes Verhalten)

  • wie Interface nur auf höherer Ebene

  • wenn man mit API Spec anfängt kann man früh Arbeitsgruppen entkoppeln
